수많은 여행 정보 앱이 "여기가 요즘 핫해요"라고 말하지만, 정작 '내 취향'과 '내 현실(지갑 사정)'에 딱 맞는 곳을 찾기는 어렵습니다. 이 서비스는 사용자가 직접 검색어조차 떠올리지 못하는 '무의식적인 진짜 취향'을 찾아내어, 당장 떠날 수 있는 가장 현실적인 목적지를 제안하는 것을 목표로 합니다.

유튜브로 무의식적으로 보던 관심사를 현실로 체험해보세요 ㅎㅎ

- **Who (누가):**  여행을 가고 싶지만 정보의 바다에서 결정 장애를 겪는 모든 사람.
- **Why (왜):**  단순 인기순 추천이 아닌, 사용자의 '숨겨진 취향(DNA)'과 '현실적 조건(비용/날씨)'을 동시에 만족하는 완벽한 매칭을 제공하기 위해.
- **What (무엇을):**  사용자의 유튜브 시청 데이터, 글로벌/로컬 뉴스 트렌드, 그리고 실시간 여행 경비 데이터를 융합한 **'맞춤형 국내/해외 여행지 추천 결과'**를.
- **Where (어디로):**  예산과 일정에 맞춰 전 세계 도시부터 국내 소도시까지 유연하게 타겟팅. (해외는 항공권, 국내는 KTX/고속버스 등 교통수단 연계)
- **When (언제):**  사용자가 여행을 떠나고자 하는 바로 '그 시기'에 맞춰. (해당 시점의 실시간 날씨, 축제 정보, 안전 지수, 성수기 비행기표 가격 등을 즉각 반영)
- **How (어떻게 - 3대 핵심 기술 아키텍처):**
    1. **개인화 (On-Demand):** YouTube Data API로 사용자의 구독/시청 기록을 분석해 핵심 취향 키워드 추출 (비용 방어를 위해 캐싱 활용).
    2. **트렌드 분석 (Batch):** News/커뮤니티 API 데이터를 하둡(Hadoop) 생태계에서 맵리듀스(MapReduce)로 주기적으로 분산 처리하여 도시별 감성 및 테마 점수 구축. → **도시별 감성 및 테마에 대한 세계지도 시각화**
    3. **현실 필터링 (Real-Time):** 추천 직전, 스카이스캐너(항공/교통), 환율, OpenWeather API 등을 호출해 예산과 안전 기준을 통과한 최종 목적지만 선별하여 프론트엔드로 전달.

### **1: 사용자 페르소나 - 유튜브 API**

**유튜브 API**를 통해 수집되는 데이터입니다. 매번 호출하면 할당량(Quota)이 부족해지므로, 사용자가 로그인을 하거나 "내 취향 분석하기" 버튼을 눌렀을 때만 작동하게 합니다.

• **프로세스:** 

1. Vue.js 프론트엔드에서 사용자가 분석 요청.
2. Spring Boot 서버가 YouTube Data API 호출.
3. 구독 목록과 최근 시청 영상의 제목/설명에서 핵심 키워드(예: `캠핑`, `사찰`, `오마카세`, `가성비`) 추출.
4. 이 키워드들을 MySQL이나 Redis에 **'사용자 여행 DNA'**로 저장 (예: 1주일간 캐싱).

### **2: 글로벌/로컬 트렌드 - 뉴스 API+여러가지 지수**

**뉴스 API, 물가지수, 안전지수** 등 방대한 텍스트와 통계 데이터를 처리하는 영역입니다. 실시간성이 1분 1초 단위로 필요하진 않으므로 하둡 생태계에서 주기적으로 돌립니다.

• **프로세스:**

1. 매일 새벽, 전 세계(혹은 국내 주요 지역) 뉴스와 커뮤니티 글을 크롤링/API로 수집.

2. Hadoop에 적재하고 MapReduce(또는 Spark)를 돌려 지역별 '현재 테마'와 '감성 점수(긍정/부정)'를 추출.
3. "이번 주 삿포로는 눈축제로 긍정 점수 최고", "파리는 시위 뉴스로 안전 점수 하락" 같은 정제된 데이터를 DB에 업데이트.

### **3: 실시간 제약 조건 - 실시간 날씨, 교통, 환율**

사용자가 앱을 켜서 **"이번 주말에 갈 국내 여행지"** 또는 **"다음 달에 갈 해외 여행지"**를 검색하는 순간 호출되는 데이터입니다.

• **프로세스:**
    1. 스카이스캐너 API(해외)나 KTX/고속버스 API(국내)로 **교통비** 조회.
    2. OpenWeather API로 해당 날짜의 **날씨** 조회.
    3. 환율 API로 실시간 **가성비** 계산.

1. **DNA 매칭 :** 유튜브와 지역별 테마의 키워드가 얼마나 일치하는가?

 (코사인 유사도 활용)

2. **트렌드 가중치:** 하둡에서 분석한 뉴스 긍정 점수가 높은가?

3. **비용 감점:** 비행기표/교통비 + 현지 물가지수가 사용자의 예산을 초과하면 점수를 대폭 깎거나 추천에서 제외.

4. **안전/날씨 보너스:** 치안이 좋고 여행 기간의 날씨가 맑으면 가산점.

**🇰🇷 국내 vs 🌐 해외 분기 처리**
사용자가 **"국내 여행"**을 선택했다면, 파이프라인이 유연하게 바뀌어야 합니다.
• **교통비:** 비행기표 대신 KTX/SRT 요금 데이터, 혹은 렌터카/주유비 추정치 API 사용.
• **트렌드:** 해외 NewsAPI 대신 네이버/다음 뉴스 API 및 국내 여행 커뮤니티 데이터 분석.
• **물가:** 글로벌 물가지수 대신 '한국관광데이터랩'의 지역별 평균 숙박비/식비 데이터 활용.
이 아키텍처라면 사용자 입장에서는 내 유튜브 취향을 정확히 저격하면서도, 내 지갑 사정과 이번 주말 날씨까지 고려해 주는 '초개인화 여행 비서'를 경험하게 될 것입니다.
