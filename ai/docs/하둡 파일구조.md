# 하둡 파일구조

## 1) 추천 원칙: “공통 컨테이너 + 도메인별 payload”

- **공통 컬럼(메타데이터)**: 모든 데이터셋이 동일하게 가지는 최소 정보
- **도메인 payload**: 뉴스/항공권/환율별로 다른 필드는 payload로 분리

예시(논리 구조):

- `event_time` (데이터가 의미하는 시간: 뉴스 발행시간, 환율 기준일, 항공권 출발일/조회시각 중 택1~2)
- `ingest_time` (수집한 시각)
- `source` (gdelt/rss/air_api/exchangerate_host 등)
- `entity` (country=KR 같은 공통 차원 / 필요하면 city, currency_pair 등)
- `payload` (도메인별 원본/정규화 필드)

이렇게 하면 **Spark/Trino/Hive에서 공통 컬럼으로 먼저 필터링**하고, 상세 분석은 payload를 펼치면 됨.

## 2) 파일 포맷 추천

- Bronze(원본 보관): `JSONL`  (원본 그대로 + 메타)
- Silver/Gold(분석용): **Parquet + Snappy** (컬럼형이라 조회/압축/비용 모두 유리)

## 3) HDFS 디렉토리 구조 추천 (가장 많이 쓰는 형태)

공통 규칙만 통일하고, 도메인별로 dataset을 나눠:

```
/data/{layer}/{domain}/
  dt=YYYY-MM-DD/
    hour=HH/          (뉴스/항공권 “수집시각” 기준이면 유용)
      part-*.parquet
```

- `layer`: `bronze | silver | gold`
- `domain`: `news | flights | fx`
- `dt/hour`는 **ingest_time(수집시간) 기준**으로 두는 걸 추천 (누락/재수집/백필에 강함)

그리고 도메인별 “조회가 잦은 키”가 있으면 **추가 파티션**도 가능:

- 뉴스: `source_country=US` 또는 `event_country=US`
- 환율: `base=KRW/quote=USD` 또는 `pair=KRW_USD`
- 항공권: `origin=ICN/dest=NRT` 또는 `route=ICN_NRT`

다만 파티션을 너무 많이 쪼개면(카디널리티 높으면) 파일이 잘게 쪼개져서 오히려 느려져.

**원칙:** “자주 WHERE로 거는 값”만 파티션, 나머지는 컬럼 인덱싱(Parquet)로 해결.

## 4) 도메인별로 “시간”을 어떻게 둘지

공통 구조를 쓰되, 시간은 2개를 분리해두는 게 안전해:

- `ingest_time`: 언제 수집했나 (파티션 기준 추천)
- `event_time`: 데이터가 의미하는 기준 시각
    - 뉴스: publish_time
    - 환율: rate_date
    - 항공권: departure_date(또는 검색시각도 별도 필드로)

## 5) 최소 공통 스키마 예시(실무에서 자주 쓰는 형태)

- `dataset` (news/flights/fx)
- `schema_version`
- `ingest_time`
- `event_time`
- `source`
- `country` (가능하면 ISO-3166-1 alpha-2/3)
- `currency` / `pair` / `route` 같은 도메인 키 (없으면 null)
- `payload` (JSON string 또는 nested struct)

---

### 결론

- *“공통 포맷(메타 규약) + 도메인별 payload”**로 가고,
- **HDFS 디렉토리는 layer/domain/dt/hour** 같은 공통 규칙으로 통일하는 게 베스트야.
- 분석용은 **Parquet**, 원본 보관은 JSONL로 분리하면 운영 난이도가 확 내려가.

원하면 내가 너희 프로젝트 기준으로(뉴스는 URL+발생국가만 있음, 항공권/환율은 어떤 API 쓰는지) **각 도메인별 Silver 스키마(컬럼명/타입) + 추천 파티션 조합**을 딱 정해줄게.

# 결정 사항

## 디렉토리 구조

`/data/layer/domain/dt/hour/...(필요하면 확장)` 

- layer : Bronze, Silver, Gold
- domain : news, airticket, exchange
- dt : 수집일 ( ex: 2026-03-04 )
- hour: 수집시각 ( ex : 05-00 )

## 파일 포맷

- 원본 보관 ( Bronze ) : JSONL
- 분석용 ( Silver / Gold ) : Parquest

## 데이터 양식

- **공통 컬럼(메타데이터) + 도메인 payload ( 필드 들 )**
- 공통 컬럼
    - `event_time` (데이터가 의미하는 시간: 뉴스 발행시간, 환율 기준일, 항공권 출발일/조회시각 중 택1~2)
    - `ingest_time` (수집한 시각)
    - `source` (gdelt/rss/air_api/exchangerate_host 등)
