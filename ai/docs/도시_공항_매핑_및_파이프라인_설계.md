# 도시 ↔ 공항 매핑 & 파이프라인 설계 변경안

---

## 1. 핵심 문제

**공항이 없는 도시** == 크롤링 불가능 → 인근 공항으로 매핑 필요

예시:
- 교토 → KIX (간사이 국제공항, 오사카)
- 인터라켄 → ZRH (취리히) 또는 GVA (제네바)
- 루체른 → ZRH (취리히)
- 바기오 → MNL (마닐라)

---

## 2. 파이프라인 설계 변경

### 기존 방식 ❌
```
도시 → 크롤링 → 공항코드 1:1
```

### 변경 방식 ✅
```
공항 단위로 크롤링 → Silver에서 city_id로 1:N 분기
```

```
[크롤링] dest_airport=KIX 1회
         ↓ Silver ETL
[city_id=OSAKA]
[city_id=KYOTO]   ← 같은 가격 데이터 재사용
```

> **핵심:** `city_id`(서비스 표시용)와 `dest_airport`(실제 크롤링 공항)를 분리

---

## 3. 파일 구조 변경

### 추가 필요 파일

**`city_airport_mapping.json`** — 공통 참조 테이블

```json
[
  {
    "city_id": "KYOTO",
    "city_name_kr": "교토",
    "country": "JP",
    "dest_airport": "KIX",
    "note": "교토 공항 없음 → 간사이 국제공항(오사카) 사용"
  },
  ...
]
```

**크롤링 대상 = 이 파일에서 `dest_airport` 기준 중복 제거한 목록**

## 6. 파이프라인 변경 요약

| 단계 | 기존 | 변경 후 |
|:---|:---|:---|
| 크롤링 기준 | 도시명 | **공항 코드** (중복 제거) |
| Bronze 저장 | city_id + dest_airport | 동일 (dest_airport 기준) |
| Silver ETL | 1:1 변환 | **1:N 분기** (공유공항 도시 복제) |
| city_id | 크롤링 도시 = 서비스 도시 | **매핑 테이블 통해 분리** |

### Silver ETL 코드 변경 포인트

```python
# city_airport_mapping.json 로드
airport_to_cities = {}
for record in mapping:
    airport = record["dest_airport"]
    if airport not in airport_to_cities:
        airport_to_cities[airport] = []
    airport_to_cities[airport].append(record["city_id"])

# Bronze 레코드 하나를 여러 city_id로 복제
for city_id in airport_to_cities[rec["entity"]["dest_airport"]]:
    silver_rec = {**rec, "entity": {**rec["entity"], "city_id": city_id}}
```
