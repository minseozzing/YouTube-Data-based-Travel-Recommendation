# ✈️ 항공권 크롤링 데이터 → 파이프라인 컬럼 정리

---

## 1. 크롤러별 수집 데이터 현황

### 🔵 Skyscanner 크롤러 (`scrape_multi_dest.py` / `scrape_multi_destmin.py`)
> **주력 데이터소스** — 날짜별 항공권 최저가 수집

| 크롤러 출력 필드 | 예시 | 용도 |
| :--- | :--- | :--- |
| `City` | `DFW`, `CTS`, `FUK` | 목적지 공항 IATA 코드 |
| `Date` | `2026-05-12` | 출발 날짜 |
| `Price` | `285000` | 해당 날짜의 최저 항공권 가격 (KRW) |

> ⚠️ **한계**: `City`가 공항 코드(DFW)라서 여행 도시(서울, 교토 등)와의 **매핑 테이블**이 필요.

---

### 🟢 Google Explore 크롤러 (`google_explore.py`)
> **보조 데이터소스** — 도시별 항공 정보 스냅샷 수집

| 크롤러 출력 필드 | 예시 | 용도 |
| :--- | :--- | :--- |
| `Destination` | `도쿄`, `방콕` | 여행 도시명 (한국어) |
| `Month` | `3월` | 탐색 월 |
| `Date` | `3월 10일 ~ 3월 17일` | 날짜 범위 |
| `Stops` | `직항`, `경유 1회` | 직항 여부 |
| `Flight Duration` | `2시간 30분` | 비행 소요 시간 |
| `Ground/Extra Time` | `1시간` | 공항~도시 이동 시간 |
| `Flight Price` | `₩350,000` | 항공권 가격 |
| `Hotel Price` | `₩95,000` | 호텔 1박 평균 가격 |
| `Image URL` | `https://...` | 도시 썸네일 (미사용 가능) |
| `Timestamp` | `2026-03-04 05:00:00` | 수집 시각 |

## 2. 실제로 쓸 데이터만 추리기

### ✅ 사용할 데이터

| 데이터 항목 | 출처 | 이유 |
| :--- | :--- | :--- |
| 날짜별 최저 항공권 가격 | Skyscanner | 달력 UI, 가격 트렌드 핵심 |
| 출발 날짜 | Skyscanner | 날짜별 조회의 기준 |
| 목적지 공항 코드 | Skyscanner | city_id 매핑에 사용 |
| 직항 여부 | Google Explore | `isDirect` 필드 |
| 비행 소요 시간 | Google Explore | `flightTime` 필드 |
| 호텔 평균 가격 | Google Explore | `avgAccommodation` 필드 |
| 수집 시각 | 둘 다 | `ingest_time` (파티션 기준) |

### ❌ 제외할 데이터

| 데이터 항목 | 이유 |
| :--- | :--- |
| `Image URL` | 서비스 API 명세에 없음 |
| `Ground/Extra Time` | 서비스 스펙에 없음 (추후 활용 가능) |
| `Date` (Google, 범위 텍스트) | 파싱 어려움, Skyscanner에서 이미 날짜 수집 |
| `Month` (Google, '3월' 텍스트) | Skyscanner 날짜에서 파생 가능 |

## 3. Bronze 저장 스키마 (JSONL)

> 크롤러가 수집 직후 HDFS에 저장하는 원본 데이터

### 3-1. Skyscanner Calendar Price (날짜별 가격)

**저장 경로**: `/data/bronze/airticket/dt=YYYY-MM-DD/hour=HH/skyscanner_calendar.jsonl`

```json
{
  "dataset": "airticket",
  "schema_version": 1,
  "source": "skyscanner",
  "ingest_time": "2026-03-04T05:00:00",
  "event_time": "2026-05-12",
  "entity": {
    "city_id": "OSAKA",
    "origin": "ICN",
    "dest_airport": "KIX"
  },
  "payload": {
    "price": 285000,
    "currency": "KRW"
  }
}
```

### 3-2. Google Explore Snapshot (도시별 항공 정보)

**저장 경로**: `/data/bronze/airticket/dt=YYYY-MM-DD/hour=HH/google_explore.jsonl`

```json
{
  "dataset": "airticket",
  "schema_version": 1,
  "source": "google_explore",
  "ingest_time": "2026-03-04T05:00:00",
  "event_time": "2026-03-04",
  "entity": {
    "city_id": "OSAKA",
    "origin": "ICN"
  },
  "payload": {
    "stops": 0,
    "flight_duration_min": 150,
    "avg_accommodation": 95000,
    "currency": "KRW"
  }
}
```

## 4. Silver 테이블 컬럼 정의

> Spark ETL로 Bronze를 정제한 분석용 테이블 (Parquet 저장)

### 4-1. `calendar_price` — 날짜별 최저 항공권

| 컬럼명 | 타입 | 설명 | 출처 |
| :--- | :--- | :--- | :--- |
| `city_id` | STRING | 여행 도시 ID (예: OSAKA) | entity.city_id |
| `origin` | STRING | 출발 공항 (ICN 고정) | entity.origin |
| `dest_airport` | STRING | 도착 공항 코드 (예: KIX) | entity.dest_airport |
| `departure_date` | DATE | 출발 날짜 | event_time |
| `price` | INT | 최저 항공권 가격 (KRW) | payload.price |
| `collected_date` | DATE | 수집 날짜 | ingest_time 날짜부분 |
| `ingest_time` | TIMESTAMP | 전체 수집 시각 | ingest_time |

### 4-2. `explore_snapshot` — 도시별 항공 정보

| 컬럼명 | 타입 | 설명 | 출처 |
| :--- | :--- | :--- | :--- |
| `city_id` | STRING | 여행 도시 ID | entity.city_id |
| `stops` | INT | 경유 횟수 (0=직항) | payload.stops |
| `flight_duration_min` | INT | 비행 시간 (분 단위) | payload.flight_duration_min |
| `avg_accommodation` | INT | 평균 숙박비 (KRW/박) | payload.avg_accommodation |
| `collected_date` | DATE | 수집 날짜 | ingest_time 날짜부분 |
| `ingest_time` | TIMESTAMP | 전체 수집 시각 | ingest_time |

## 5. Gold 테이블 컬럼 정의

> Spark Aggregation으로 Silver에서 생성하는 API 서빙용 테이블

### 5-1. `calendar_month_latest` — 달력 월별 최신 가격 (API: `/api/air/calendar-prices`)

| 컬럼명 | 타입 | 설명 |
| :--- | :--- | :--- |
| `city_id` | STRING | 여행 도시 ID |
| `departure_date` | DATE | 출발 날짜 |
| `price` | INT | 최신 수집 최저가 |
| `status` | STRING | LOW / MID / HIGH (월 평균 대비) |
| `collected_date` | DATE | 기준 수집 날짜 |

### 5-2. `flight_summary_latest` — 도시 항공 요약 (API: `/api/air/flight-summary`)

| 컬럼명 | 타입 | 설명 |
| :--- | :--- | :--- |
| `city_id` | STRING | 여행 도시 ID |
| `flight_time` | STRING | 비행 시간 (예: "2시간 30분") |
| `avg_accommodation` | INT | 평균 숙박비 (KRW) |
| `is_direct` | BOOLEAN | 직항 여부 |
| `collected_date` | DATE | 기준 수집 날짜 |

### 5-3. `price_trend_30d` — 월별 30일 가격 추이 (API: `/api/air/price-trends`)

| 컬럼명 | 타입 | 설명 |
| :--- | :--- | :--- |
| `city_id` | STRING | 여행 도시 ID |
| `target_month` | STRING | 조회 기준 월 (YYYY-MM) |
| `collected_date` | DATE | 수집 날짜 |
| `price` | INT | 해당 수집일의 최저가 |
| `recommendation` | STRING | BUY / WAIT |

### 5-4. `round_trip_best` — 월별 최저 왕복 (API: `/api/air/low-price`)

| 컬럼명 | 타입 | 설명 |
| :--- | :--- | :--- |
| `city_id` | STRING | 여행 도시 ID |
| `target_month` | STRING | 조회 기준 월 (YYYY-MM) |
| `lowest_price` | INT | 최저 항공권 가격 |
| `departure_date` | DATE | 최저가 출발일 |
| `return_date` | DATE | 복귀일 (출발 +3~7일 추정) |
| `collected_date` | DATE | 기준 수집 날짜 |

## 6. 아직 필요한 것 (미확정)

| 항목 | 설명 | 우선순위 |
| :--- | :--- | :--- |
| **도시 → 공항 매핑 테이블** | 교토→KIX, 뉴욕→JFK 등 100개 도시 기준 | 🔴 높음 (크롤러 실행 전에 필요) |
| **`city_id` 통일 방식** | 팀 공통 ID 체계 결정 필요 (IATA? 자체?) | 🔴 높음 |
| **`status` 판단 기준** | LOW/MID/HIGH 분위 기준 정의 필요 | 🟡 중간 |
| **`return_date` 계산 방법** | 왕복 최저 조합 계산 로직 | 🟡 중간 |
