# UI Structure Analysis: 로그인 페이지

## 1. Layout Pattern
2-column 분할 (좌: 텍스트+CTA 약 55% / 우: 이미지 카드 약 45%). 전체 너비 상단 네비게이션. 전체 페이지 레이아웃. 모달/오버레이 없음.

## 2. Estimated Grid Structure
12컬럼 기반 추정 (약 7+5 또는 6+6 분할). 좌측 내부 하단 Feature 섹션은 2-column 균등 분할. 상단 네비게이션은 전체 너비. 우측 패널은 이미지 단일 채우기. 명시적 거터 없음(좌우 패널 edge-to-edge 분할).

## 3. Visual Hierarchy Order
- Tier 1: 좌측 메인 헤드라인 "다음 여행을 계획해 보세요." — 최대 폰트, 고중량, 흑색 고대비, 좌상단 시선 진입점
- Tier 2: 우측 이미지 카드 패널 전체 — 고채도 자연 사진 + 오버레이 텍스트 "프리미엄 여행의 시작"
- Tier 3: Google 계정으로 계속하기 버튼 — 테두리+로고 아이콘, 메인 CTA
- Tier 4: 이메일 입력 필드 및 하단 텍스트 링크
- Tier 5: 하단 Feature 카드 2개 ("빠르고 간편함", "실시간 도우미")
- Tier 6: 상단 네비게이션 링크들
- Tier 7: 최하단 푸터 텍스트

## 5. Interaction Elements
| Element | Type | Likely Trigger |
|---|---|---|
| Google 계정으로 계속하기 버튼 | button | Google OAuth 로그인 URL 호출 → 리다이렉트 |
| 이메일 입력 필드 | text input | 이메일 기반 로그인 폼 입력 |
| 네비게이션 링크들 | link | 페이지 이동 |
| 플래너지로 돌아가기 | link | 이전 페이지 이동 |

## 6. Data Display Patterns
| Area | Display Type | Dynamic Data |
|---|---|---|
| 메인 헤드라인 | text | No |
| 우측 이미지 | image | No |
| 아바타 그룹 | image-list | No |
| Feature 카드 | card-list | No |

## 7. Repeated Patterns
Feature 카드: 아이콘 + 라벨 + 설명 텍스트 구조 2회 반복 (수평 2열).

## 8. Dominant Element & Information Density
- **Dominant**: 좌측 메인 헤드라인 "다음 여행을 계획해 보세요." — 최대 폰트 크기 + 고중량 + 흑색 고대비 + 좌상단 F-패턴 위치
- **Density**: Medium — 좌측은 헤드라인/CTA/Feature 카드 수직 적층이나 여백 확보, 우측은 이미지 단순 채우기. 전체 평균 Medium.
