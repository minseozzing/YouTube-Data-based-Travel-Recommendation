# UI Structure Analysis: 저장된 도시 상세 리포트

## 1. Layout Pattern
top-nav + hero + dashboard. 전체 너비 네비게이션 + 전체 너비 히어로 섹션(도시 사진 배경) + 하단 2열 카드 대시보드(좌: 항공권+이슈 / 우: 환율+물가). 전체 페이지 레이아웃.

## 2. Estimated Grid Structure
단일 중앙 정렬 고정폭 컨테이너. 히어로 섹션: 전체 너비 1열. 하단 콘텐츠: 비대칭 2열 (좌 약 55% / 우 약 45%). 각 열에 카드 2개씩 수직 배치. 카드 간 거터 균등 (4px/8px 기반 추정).

## 3. Visual Hierarchy Order
- Tier 1: 히어로 섹션 "시드니, 호주" 텍스트 — 최대 폰트, 흰색 고대비, 전체 너비 이미지 배경 좌상단
- Tier 2: 히어로 우상단 원형 점수 배지 (예: "98점") — 고채도 주황/노랑 + 원형 배경
- Tier 3: "982,500원" 항공권 가격 — 카드 내 최대 폰트, 굵은 웨이트
- Tier 4: "892.41원" 환율 수치 — 파란 강조 색상
- Tier 5: 섹션 헤더 레이블 4개 (항공권 가격, 환율, 주요 이슈, 해외 물가)
- Tier 6: 이슈 항목 본문, 물가 항목 리스트
- Tier 7: 하단 버튼, 네비게이션, 푸터

## 6. Data Display Patterns
| Area | Display Type | Dynamic Data |
|---|---|---|
| 도시 히어로 배경 이미지 | image | Yes |
| 매칭 점수 배지 | stat-number | Yes |

## 7. Repeated Patterns
물가 항목 행: 카테고리 아이콘 + 레이블 + 가격 수치 구조 반복 (교통, 식사, 숙박 등). 뉴스 이슈 항목: 아이콘 + 텍스트 구조 3회 반복.

## 8. Dominant Element & Information Density
- **Dominant**: 히어로 "시드니, 호주" 텍스트 — 전체 너비 이미지 배경 위 최대 폰트 + 흰색 고대비 + 좌상단 앵커 포지션
- **Density**: Medium — 4개 카드에 데이터 분산, 카드 간 여백 유지. 뉴스 카드 내부는 텍스트 밀집으로 High에 가까움
