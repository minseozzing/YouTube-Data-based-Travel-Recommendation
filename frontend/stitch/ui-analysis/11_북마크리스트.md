# UI Structure Analysis: 저장된 도시 리스트 (북마크)

## 1. Layout Pattern
top-nav + content. 전체 너비 네비게이션 + 브레드크럼 + 헤더 텍스트 + 4-column 카드 그리드 + 페이지네이션 + 푸터. 전체 페이지 레이아웃. 모달/오버레이 없음.

## 2. Estimated Grid Structure
12컬럼 기반 추정 (카드 그리드: 4열, 각 3컬럼). 카드 간 거터 약 16–20px 균등. Row 1: 카드 4개 완전 채움. Row 2: 카드 1개 + "도시 추가하기" 플레이스홀더 1개 + 빈 슬롯 2개. 헤더/브레드크럼/푸터: 전체 컨테이너 너비 단일 컬럼.

## 3. Visual Hierarchy Order
- Tier 1: "총 저장한 도시 : 12개" 헤드라인 — 최대 본문 텍스트, 파란 강조 숫자 "12개", 콘텐츠 좌상단
- Tier 2: 도시 카드 사진들 (파리, 도코, 런던, 뉴욕, 포르투) — 대형 이미지 블록, 고대비
- Tier 3: 카드 하단 도시명 레이블 — 중간 볼드 타이포그래피
- Tier 4: 대륙/지역 배지 (EUROPE, ASIA, AMERICAS) — 컬러 배지, 소형
- Tier 5: 카드 설명 텍스트 — 소형, 낮은 가중치, 회색
- Tier 6: 날짜 스탬프 — 최소 텍스트, 최저 대비
- Tier 7: "도시 추가하기" 플레이스홀더 — 점선 테두리, 중앙 정렬 플러스 아이콘
- Tier 8: 상단 네비게이션 바
- Tier 9: 페이지네이션 (1, 2, 3, 화살표)
- Tier 10: 푸터

## 5. Interaction Elements
| Element | Type | Likely Trigger |
|---|---|---|
| 북마크 카드 | clickable card | 북마크 상세 리포트 페이지 이동 |
| "도시 추가하기" 플레이스홀더 | button | 메인 페이지(추천) 이동 또는 검색 모달 |
| 필터 드롭다운 | dropdown | 대륙/지역별 필터링 |
| 페이지네이션 버튼 | button | 페이지 전환 |
| 네비게이션 링크 | link | 페이지 이동 |

## 6. Data Display Patterns
| Area | Display Type | Dynamic Data |
|---|---|---|
| 북마크 카드 그리드 | image-grid | Yes (GET /api/bookmarks) |
| 총 저장 도시 수 | stat-number | Yes |
| 대륙 배지 | badge/tag | Yes |
| 날짜 스탬프 | text | Yes |

## 7. Repeated Patterns
북마크 카드: 도시 사진 + 도시명 + 대륙 배지 + 설명 + 날짜 구조가 Row 1에서 4회, Row 2에서 1회 총 5회 반복. 동일 컴포넌트 `BookmarkCard`로 구현 가능.

## 8. Dominant Element & Information Density
- **Dominant**: 도시 카드 이미지 그리드 (집합적) — 최대 면적, 고채도 사진. 개별적으로는 파리 카드(좌상단, F-패턴 진입점)
- **Density**: Medium — 카드 그리드가 뷰포트 약 55–60% 점유, 외곽 마진과 Row 2 빈 슬롯으로 여백 확보
