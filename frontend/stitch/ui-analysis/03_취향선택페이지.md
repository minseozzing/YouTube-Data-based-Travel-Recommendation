# UI Structure Analysis: 취향선택 페이지

## 1. Layout Pattern
수평 스토리보드 레이아웃 (5개 패널이 캔버스에 가로로 배열). 단일 스크린이 아닌 UX 플로우 기획서 형태. 패널별 레이아웃 상이:
- Panel 1 (라이트 모달): 2-column 카드 그리드 내부
- Panel 2 (라이트 모달): 단일 컬럼 수직 스택
- Panel 3 (다크 전체화면): 사이드바 + 태그 그리드 영역
- Panel 4 (플로우차트): 자유 배치 다이어그램
- Panel 5 (다크 앱 화면 + 주석): 앱 목업 + 텍스트 2-split

## 2. Estimated Grid Structure
전체 캔버스에 통합 그리드 없음. 패널별 독립 그리드 사용. Panel 3(다크 화면)은 사이드바+콘텐츠 영역 추정 2컬럼. 태그 칩들은 flex wrap 배열. Panel 1의 CTA 카드는 2-column 균등 그리드. Panel 4(플로우차트)는 중앙 수직축 기준 좌우 분기 구조.

## 3. Visual Hierarchy Order
- Tier 1: Panel 3 (다크 전체화면) — 가장 넓은 면적 + 최고 대비(네이비 배경 vs 밝은 캔버스)
- Tier 2: Panel 5 (다크 앱 화면 + 주석 블록) — 두 번째 면적, 어두운 배경
- Tier 3: Panel 4 (플로우차트) — 중간 크기, 박스/화살표 구조
- Tier 4: Panel 1 (라이트 모달) — 중간 크기, 낮은 대비
- Tier 5: Panel 2 (라이트 모달) — 가장 작은 면적, 최저 시각 가중치
- Tier 6: 우측 주석 텍스트 블록

## 5. Interaction Elements
| Element | Type | Likely Trigger |
|---|---|---|
| 태그 칩 (Panel 3) | toggle chip | 태그 선택/해제 |
| CTA 버튼 (Panel 1, 하단 파란 버튼) | button | 선호도 저장 후 다음 단계 이동 |
| YouTube 연동 선택 (Panel 4 분기) | button/link | YouTube 연동 동의 URL 발급 |
| 직접 선택 분기 (Panel 4) | button | 태그 직접 선택 화면으로 이동 |

## 6. Data Display Patterns
| Area | Display Type | Dynamic Data |
|---|---|---|
| 태그 칩 목록 (Panel 3) | badge/tag | Yes (서버 태그 목록) |
| YouTube 연동 상태 (Panel 5) | status indicator | Yes |
| 플로우차트 (Panel 4) | decoration | No |

## 7. Repeated Patterns
태그 칩: 동일 크기/스타일의 칩 요소가 Panel 3에서 다수 반복 (flex wrap). Panel 1 CTA 카드: 아이콘+텍스트 카드 2회 반복.

## 8. Dominant Element & Information Density
- **Dominant**: Panel 3 (다크 태그 선택 화면) — 최대 면적 + 최고 명암비(네이비 배경) + 최고 내부 밀도 + 하단 파란 CTA 버튼
- **Density**: 캔버스 전체 Low / Panel 3 내부 High / Panel 4 내부 Low / 나머지 Medium
