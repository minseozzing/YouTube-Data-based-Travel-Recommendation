# UI Structure Analysis: 추천이유 모달

## 1. Layout Pattern
modal-overlay. 배경 페이지 위 중앙 모달. 모달 내부: 비대칭 2열 (좌: 목적지 카드 약 30% / 우: 탭+콘텐츠 약 70%). 활성 탭: "추천 이유".

## 2. Estimated Grid Structure
모달 내부: 명시적 12컬럼 없음. 좌우 비대칭 2-zone 분할 (고정 좌 패널 + 유동 우 콘텐츠). 우측 콘텐츠 내부: 단일 컬럼 수직 스택 (탭 바 → AI 분석 섹션 → 본문 → 키워드 태그 → 관련 정보 목록). 하단 관련 정보 카드 3개: 단일 컬럼 수직 배치.

## 3. Visual Hierarchy Order
- Tier 1: 모달 컨테이너 전체 — 중앙 위치, 대형 면적, 딤드 배경 대비
- Tier 2: 좌측 목적지 카드 (레이크 루이스 사진 + "레이크 루이스" 텍스트 + 파란 "저장하기" 버튼)
- Tier 3: 탭 네비게이션 행 (추천 이유 / 생활물가 비교 / 항공권 탐색 / 현지 뉴스)
- Tier 4: "AI 맞춤 추천 분석" 섹션 헤더 + 파란 아이콘
- Tier 5: 본문 텍스트 (파란색 강조 텍스트 "38%의 특성을" 포함)
- Tier 6: 키워드 태그 필 (인기 키워드, #레이크루이스 등)
- Tier 7: 관련 정보 목록 3개 (생활물가 비교 / 최저 항공권 / 최근 현지 소식)
- Tier 8: 배경 페이지 네비게이션 바 (딤드)

## 5. Interaction Elements
| Element | Type | Likely Trigger |
|---|---|---|
| "저장하기" 버튼 | button | 북마크 생성 API 호출 |
| 탭 (추천이유/생활물가/항공권/현지뉴스) | tab | 탭 콘텐츠 전환 |
| 관련 정보 카드 (생활물가 비교) | clickable card | 생활물가 탭으로 전환 |
| 관련 정보 카드 (최저 항공권) | clickable card | 항공권 탭으로 전환 |
| 관련 정보 카드 (현지 소식) | clickable card | 현지 뉴스 탭으로 전환 |
| 모달 닫기 | button | 모달 닫기 |

## 6. Data Display Patterns
| Area | Display Type | Dynamic Data |
|---|---|---|
| 목적지 사진 + 이름 | image + text | Yes (cityId 기반) |
| AI 분석 본문 | text | Yes |
| 키워드 태그 | badge/tag | Yes |
| 관련 정보 요약 | card-list | Yes |

## 7. Repeated Patterns
관련 정보 아이템: 아이콘 + 레이블 + 설명 텍스트 구조 3회 반복 (수직 단일 컬럼). 탭 아이템: 동일 스타일 탭 레이블 4회 반복.

## 8. Dominant Element & Information Density
- **Dominant**: 좌측 목적지 사진 카드 — 전체 컬럼 높이 점유, 고채도 사진, 대형 흰색 타이포그래피 + 파란 CTA 버튼
- **Density**: Medium — 우측 콘텐츠에 섹션 여러 개지만 여백 유지. 좌측은 이미지 단순 채우기.
