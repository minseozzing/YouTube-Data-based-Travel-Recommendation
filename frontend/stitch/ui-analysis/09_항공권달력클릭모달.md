# UI Structure Analysis: 항공권 정보 모달 - 달력 날짜 선택

## 1. Layout Pattern
modal-overlay (2중 레이어). 배경 페이지 위 기본 모달 + 기본 모달 내 차트 영역 위 플로팅 날짜 선택 시트(2차 모달). 기본 모달 내부: 비대칭 2열 (좌: 목적지 카드 / 우: 탭+차트+항공편 목록). 2차 시트: 우측 차트 영역 중앙에 플로팅 배치.

## 2. Estimated Grid Structure
기본 모달: 좌우 비대칭 2-zone 분할 (약 30:70). 우측 콘텐츠: 단일 컬럼 수직 스택. 항공편 목록 행: 암묵적 5–6컬럼 (항공사 로고/이름/출발-도착 시간/소요시간/가격). 플로팅 시트: 우측 컬럼 중앙 부유, 그리드 기준 이탈.

## 3. Visual Hierarchy Order
- Tier 1: 플로팅 날짜 선택 시트 ("12월 18일 항공편") — 최상위 z-layer, 흰색 카드, 딤드 배경 대비
- Tier 2: 좌측 목적지 카드 (시드니 사진 + 헤딩 + 파란 CTA)
- Tier 3: 가격 트렌드 차트 + 파란 가격 배지 ("Cheaper!")
- Tier 4: 탭 네비게이션 행
- Tier 5: 항공편 목록 행 (항공사명, 시간, 가격)
- Tier 6: 상단 네비게이션 바
- Tier 7: 푸터

## 5. Interaction Elements
| Element | Type | Likely Trigger |
|---|---|---|
| "저장하기" 버튼 | button | 북마크 생성 API 호출 |
| 탭 (추천이유/생활물가/항공권/현지뉴스) | tab | 탭 콘텐츠 전환 |
| 차트 날짜 포인트 클릭 | clickable chart point | 날짜별 항공편 시트 오픈 |

| 플로팅 시트 닫기(X) | button | 시트 닫기 |

## 6. Data Display Patterns
| Area | Display Type | Dynamic Data |
|---|---|---|
| 가격 트렌드 라인 차트 | chart (line) | Yes |
| 항공편 목록 행 | table | Yes |
| 날짜별 항공편 시트 | card-list | Yes |
| 가격 배지 | badge/tag | Yes |

## 7. Repeated Patterns
항공편 행: 항공사 로고 + 이름 + 출발/도착 시간 + 소요시간 + 가격 구조 3회 이상 반복 (항공편 목록 + 플로팅 시트 내 동일 패턴).

## 8. Dominant Element & Information Density
- **Dominant**: 플로팅 날짜 선택 시트 — 최상위 z-layer + 흰색 카드 + 딤드 배경 + 인터랙션 포커스
- **Density**: Medium — 우측에 차트+목록 집중. 플로팅 시트가 차트를 가리며 레이어 복잡도 추가.
