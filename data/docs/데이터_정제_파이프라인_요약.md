# ✈️🏨 항공권 & 숙박 데이터 정제 방식 요약

## 전체 파이프라인 흐름

```
크롤러 (수집)
    ↓
Bronze Layer — HDFS에 JSONL 원본 저장
    ↓
Spark ETL (bronze_to_silver_flight.py)
    ↓
Silver Layer — HDFS에 Parquet 정제 저장
    ↓
MySQL (flight_summary 테이블) — 백엔드 서비스 DB
```

## 1. 크롤러 (Bronze 수집)

### 📍 google_flight_scraper.py (메인 데이터소스)

`data/google_flight/google_flight_scraper.py`

Google Flights Explore 페이지에서 도시별로 다음 데이터를 수집:

| 수집 항목 | 예시 | 설명 |
|---|---|---|
| `city_id_str` | `TOKYO` | 내부 도시 코드 |
| `city_name_kr` | `도쿄` | 한국어 도시명 |
| `year_month` | `2026-05` | 대상 연월 |
| `stops` | `0` (직항), `1` (경유 1회) | 경유 횟수 |
| `flight_duration` | `165` | 비행 시간 (분) |
| `hotel_price` | `95000` | 평균 숙박비 (KRW/박) |
| `peak_season_months_list` | `[7,8]` | 성수기 월 목록 |
| `off_season_months_list` | `[1,2]` | 비수기 월 목록 |
| `collected_at` | `2026-03-11 10:00:00` | 수집 시각 |

**저장 경로**: `HDFS /data/bronze/airticket/google_flights/dt=날짜/hour=시간/all_cities.jsonl`

> ⚠️ **항공권 실시간 날짜별 가격(`avg_flight_price`)은 현재 미수집 상태**  
> → 스카이스캐너 등 캘린더 가격 크롤러 추가 연동 예정

## 2. Spark ETL: Bronze → Silver 정제

### 📍 bronze_to_silver_flight.py

`data/spark_pipeline/bronze_to_silver_flight.py`

### 정제 과정

1. **필터링**: `route_type == "explore_monthly_snapshot"` 데이터만 추출
2. **컬럼 변환**: 문자열 → 숫자 타입 캐스팅 (가격, 비행 시간 등)
3. **중복 제거**: 같은 `city + year_month` 기준으로 **가장 최신 수집분 1건만** 유지
4. **Silver 저장**: `ingest_date` 파티션으로 Parquet 저장

### Silver 최종 컬럼

| 컬럼명 | 타입 | 설명 |
|---|---|---|
| `city_id_str` | STRING | 도시 코드 (MySQL city 테이블 조인에 사용) |
| `year_month` | STRING | 대상 연월 (YYYY-MM) |
| `origin_airport` | STRING | 출발 공항 (ICN) |
| `stops` | INT | 경유 횟수 |
| `flight_duration` | INT | 비행 소요 시간 (분) |
| `avg_flight_price` | INT | 평균 항공권 가격 (현재 NULL) |
| `avg_hotel_price` | INT | 평균 숙박비 (KRW) |
| `peak_month_list` | STRING | 성수기 월 (콤마 구분, 예: `7,8`) |
| `off_month_list` | STRING | 비수기 월 (콤마 구분, 예: `1,2`) |
| `flight_collected_date` | TIMESTAMP | 항공 데이터 수집 시각 |
| `hotel_collected_date` | TIMESTAMP | 숙박 데이터 수집 시각 (현재 NULL) |

## 3. MySQL 적재 (JDBC Upsert)

### 적재 방식 (2단계 Upsert)

```
1. Silver DataFrame
       ↓
2. stg_flight_summary (임시 테이블) — overwrite
       ↓
3. REPLACE INTO flight_summary SELECT ... FROM stg_flight_summary
```

> Spark JDBC는 INSERT만 지원하기 때문에 **임시 스테이징 테이블 → REPLACE INTO** 방식으로 upsert를 구현합니다.

### 도시 ID 매핑 방법

Silver 데이터의 `city_name_kr` (예: `도쿄`)와 MySQL `city` 테이블의 `city_name`을 **JOIN**해서 `city.id` (FK)로 변환 후 적재합니다.

## 4. 백엔드 API 서빙

MySQL `flight_summary` 테이블에 적재된 데이터를 JPA(`FlightSummaryRepository`)로 조회하여 API 제공:

| API | 설명 |
|---|---|
| `GET /api/flights/trend/{cityId}` | 최근 6개월 가격 등락 추이 |
| `GET /api/cities/{cityId}/summary` | 도시 요약 카드 (가격, 직항, 비행시간) |
| `GET /api/flights/calendar/{cityId}` | 월별 캘린더 가격 히스토리 (MongoDB) |
